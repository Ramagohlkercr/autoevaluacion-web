# Autoevaluación WEB

Repositorio de las actividades prácticas de la materia. Cada actividad se
desarrolla en su propia branch y se integra a `main` mediante Pull Request.

## Contenido del repositorio

| Archivo | Descripción |
|---------|-------------|
| `index.html` | Noticia trabajada en las Actividades 02 y 03 (estructura semántica + formulario de suscripción). |
| `styles.css` | Hoja de estilos de `index.html`. |
| `noticia-bbc.html` | Réplica de una noticia real para la Actividad 04. |
| `noticia-bbc.css` | Hoja de estilos externa que aproxima el diseño de la noticia original. |
| `noticia-responsive.html` | Réplica de una noticia real con diseño responsive para la Actividad 05. |
| `noticia-responsive.css` | Hoja de estilos externa con media queries (desktop/tablet/mobile). |

## URLs de las noticias originales utilizadas

Estas son las fuentes de referencia empleadas en las distintas actividades para
comparar el trabajo realizado con el sitio original:

- **Actividad 04** — *SpaceX's stock market blast-off could be Musk's biggest gamble yet* (BBC News, Business):
  https://www.bbc.com/news/articles/cy8d9e4lzv1o
- **Actividad 05** — *Could humanoid robots be heading for the battlefield?* (BBC News, Technology of Business):
  https://www.bbc.com/news/articles/cedpxwe26l1o

> Las Actividades 02 y 03 utilizaron una noticia de ejemplo de elaboración propia
> ("Diario Digital") para practicar estructura HTML y formularios.

## Actividades

- **Actividad 02:** página base de la noticia en HTML.
- **Actividad 03:** HTML semántico (`nav`, `main`, `article`, `section`), formulario de suscripción y estilos CSS.
- **Actividad 04:** réplica de una noticia real de un diario online con CSS externo, aproximando su apariencia visual.
- **Actividad 05:** réplica de una noticia real con diseño responsive (media queries para desktop, tablet y mobile) y uso del Box Model.

## Flujo de trabajo con Git

1. Crear una branch específica por actividad (por ejemplo `actividad-04`).
2. Realizar los cambios y commits descriptivos.
3. Subir la branch al repositorio remoto.
4. Abrir una Pull Request hacia `main`.
5. Revisar, hacer el merge y verificar la integración.
